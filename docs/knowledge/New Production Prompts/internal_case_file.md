# Internal Case File — Formal Documentation

Generate a formal case file for UNI G&P. This document is NEVER shown to the user. It is saved to the session folder. Write in English only.

You receive the FULL conversation — no compression. Extract ALL relevant information, including framework connections you identified but did not share with the user (especially relevant for worker sessions, where responses are kept simple but internal documentation must be comprehensive).

---

# Instructions

**Structure the case file as follows:**

## 1. Reporter Information
- Name: [or "Anonymous"]
- Union / Organization: [or "Not provided"]
- Position / Role:
- Contact: address, phone, email [any available]
- Organization size:
- Country:

## 2. Company Information
- Company name:
- Site location:
- Sector: [printing / publishing / packaging / tissue / security printing / other]
- FSC certification: [yes / no / unknown]
- Multinational status: [yes — specify parent / no]
- Sustainability commitments: [code of conduct, reports, certifications]
- Number of affected workers:

## 3. Violation Classification
- Primary type: [freedom of association / collective bargaining / forced labour / child labour / discrimination / OSH / other]
- Date violation began:
- Ongoing: [yes / no / unclear]

## 4. Applicable International Frameworks
### ILO Conventions
For each applicable convention: number, title, specific relevance to this case.

### OECD Guidelines
Applicable chapters, specific provisions, due diligence steps the company failed to follow.

### FSC Standards
Applicable principles and requirements, if the company is FSC-certified.

### EU CSDDD
Scope assessment, applicable obligations, enforcement status.

### UN Guiding Principles
Applicable pillars and principles.

## 5. Chronological Account
Detailed narrative of what the company did, in what order, based on what the user described. Include dates where available. Distinguish between facts the user stated and inferences you drew.

## 6. Impact on Workers
Concrete effects: dismissals, wage loss, physical harm, psychological impact, restrictions on organizing, deterioration of working conditions.

## 7. Actions Taken by Workers / Union
Internal grievances, negotiations, protests, organizing activity, communications with management.

## 8. Legal Complaints and Other Escalations
Any complaints filed — with which bodies, when, current status. Note if documents were mentioned but not uploaded.

## 9. Risk Assessment
- Threats to life, health, or physical safety: [describe]
- Retaliation experienced: [describe]
- Retaliation feared: [describe]
- Urgency level: [immediate / high / moderate / low]

## 10. Company Context
- Local management contacts: [if known]
- Headquarters contacts: [if known]
- Major customers: [especially multinationals — CSDDD-scope companies]
- Investors / parent company: [if known]
- Supply chain position: [manufacturer / supplier / subcontractor / brand]

## 11. Evidence
- Documentary evidence: [list what exists]
- Witnesses: [number, willingness to testify]
- Documents shared during session: [list, with key information extracted]
- Evidence quality: [strong / adequate / weak / absent]
- Critical evidence gaps:

## 12. Prior History
Previous violations at this company or site, if described by the user.

## 13. Internal Assessment
### Case Strength
[Strong / Moderate / Weak / Insufficient] — with justification.

### Discrepancy Notes
Any differences between the user's account and your assessment of the situation.

### Recommended Mechanism
Which international mechanism is most appropriate: OECD NCP complaint, ILO CFA submission, FSC complaint, CSDDD enforcement, or combination.

### Strategic Notes
Political context, sector dynamics, campaign potential, precedent value, any considerations relevant to UNI's decision-making.

### Session Integrity Flag
[Normal / Low concern / High concern] — with explanation if not normal.

### Recommended Priority
[Urgent / High / Standard / Low / Refer nationally] — with justification.

**Completeness requirement:** Extract every relevant detail from the conversation. A case file that omits information the user provided is a failure. When in doubt, include it.
