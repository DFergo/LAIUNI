# Evidence Summary Instructions

The user has uploaded a document during the conversation. Summarize its contents for injection into the active context. The summary will be added to the conversation as a system-level note — the model uses it to inform responses and include in the case file.

---

# Rules

1. **Extract all relevant facts:** names, dates, locations, company names, figures, obligations, commitments, violations described, evidence of actions.

2. **Note the document type:** termination letter, union communication, company memo, audit report, legal filing, email exchange, news article, contract excerpt, code of conduct, sustainability report, other.

3. **Identify framework connections:** If the document contains content relevant to ILO conventions, OECD provisions, FSC requirements, CSDDD obligations, or company sustainability commitments, note the connection explicitly.

4. **Quote key passages** that constitute direct evidence — a sentence from a termination letter, a clause from a code of conduct, a finding from an audit report. Keep quotes short and relevant.

5. **Note contradictions or gaps:** If the document contradicts something the user said, or if it raises new questions, flag this.

6. **Note what the document does NOT contain** that might have been expected given its type (e.g., a termination letter that does not state a reason for dismissal).

7. **Format:** Structured prose. Headings: Document Type, Key Facts, Framework Relevance, Key Quotes, Notes.

8. **Length:** Concise. The summary should be significantly shorter than the original document while capturing all relevant content.
